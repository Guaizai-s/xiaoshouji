# 小手机 - 本地化沉浸式 AI 聊天 PWA

小手机是一个基于 Vue 3 的本地化 AI 多角色聊天应用，界面风格参考微信与手机桌面，主打「像真的在手机里和角色聊天」的沉浸体验。

项目当前以纯前端本地应用为主：角色、聊天记录、API 配置、表情包、钱包流水等数据默认保存在浏览器 IndexedDB 中，不依赖后端账号系统。用户可以自行配置 OpenAI 兼容接口或 Anthropic 格式接口，让不同角色接入不同模型。

## 当前定位

小手机不是普通的聊天框壳子，而是一个面向角色陪伴、剧情互动和本地私有数据管理的 AI 小世界。

它目前重点在这些方向上打磨：

- 仿微信聊天与短信风格聊天界面
- 多角色、多 API 方案、本地上下文管理
- 表情包、语音、红包、转账等特殊消息格式
- 本地钱包系统与娱乐记账
- 角色设定、用户人设、长期记忆、时间感知
- PWA 安装体验与移动端 UI 细节

## 功能特性

### 聊天与角色

- 多 AI 角色管理
- 角色头像、设定、系统提示词、聊天背景
- 每个角色可绑定独立 API 方案
- 支持上下文轮数配置
- 支持微信聊天页与短信风格聊天页
- 消息列表、通讯录、角色详情等基础手机应用结构

### API 与模型

- 支持 OpenAI 兼容接口
- 支持 Anthropic 格式接口
- 支持自定义 Base URL、模型名、API Key
- 支持流式输出开关
- API 请求带有超时与简单重试逻辑
- 可接入官方 API 或第三方中转 API

### 多媒体与特殊消息

- 图片发送与图片识别
- 语音消息生成，当前使用 MiniMax TTS
- 表情包系统与角色表情包库绑定
- 模型可通过特殊格式触发表情、语音、红包、转账等消息
- 特殊格式解析逻辑已逐步统一到 parser / directives 结构中

当前支持的模型输出格式包括：

```text
[语音:文本]
[表情:表情名]
[红包:金额:备注]
[转账:金额:备注]
```

### 本地钱包系统

小手机内置本地娱乐钱包系统，用于模拟聊天中的红包和转账。

- 用户与每个角色都有独立本地余额
- 支持用户向角色发送红包 / 转账
- 支持角色通过模型指令向用户发送红包 / 转账
- 余额不足时会阻止交易
- 交易流水保存在本地数据库
- 删除聊天消息不会回滚历史账务
- 金额使用“分”为单位进行整数存储，避免浮点误差

注意：钱包系统仅用于本地娱乐记账，不涉及真实支付。

### 时间感知与记忆

- 支持当前时间与消息时间感知
- 支持角色核心设定与长期记忆
- 支持用户人设卡与用户状态
- 支持按角色上下文构建对话
- 后续计划接入记忆库搜索、世界书与更细的预设系统

### 本地存储与备份

所有核心数据默认保存在浏览器 IndexedDB 中，包括：

- roles：角色配置
- conversations：会话列表
- messages：聊天消息
- apiProfiles：API 方案
- userPersonas：用户人设
- stickers：表情包
- assets：图片等资源
- walletAccounts：钱包账户
- walletTransactions：钱包流水

项目支持备份导出 / 导入，方便迁移本地数据。

### PWA 与移动端体验

- 支持 PWA 安装
- 适配移动端浏览器
- 支持安全区与键盘弹出布局优化
- 细化聊天列表、气泡、顶部栏、输入框等 UI
- 支持夜间模式与主题同步

## 使用说明

### 1. 配置 API 方案

在应用设置中添加 API 方案，填写：

- API 格式：OpenAI 兼容或 Anthropic
- API Key
- API 端点 Base URL
- 模型名称

OpenAI 兼容接口建议填写到 `/v1` 或根域名，项目会根据配置自动拼接聊天接口路径。

例如：

```text
https://api.example.com/v1
```

项目会请求：

```text
/v1/chat/completions
```

Anthropic 格式会请求：

```text
/v1/messages
```

### 2. 创建角色

在通讯录中创建角色，并配置：

- 角色名称
- 头像
- 系统提示词
- API 方案
- 聊天背景
- 上下文长度
- MiniMax 语音参数
- 表情包库绑定
- 核心设定与长期记忆

### 3. 开始聊天

创建角色后，可以在微信聊天页或短信页中与角色对话。

支持：

- 文字聊天
- 图片消息
- AI 图片理解
- 语音消息
- 表情包
- 红包 / 转账
- 上下文记忆

### 4. 备份与恢复

应用设置中可以导出本地备份，也可以导入备份恢复数据。

旧版本备份缺少新表时，应按空表兼容处理。

## 提示词与特殊格式框架

项目当前将提示词分为几类：

- 基础系统规则
- 时间感知
- 角色原始设定
- 核心设定
- 长期记忆
- 用户信息
- 用户状态
- 特殊消息格式
- 可用表情包列表
- 最近聊天上下文

特殊消息格式用于让模型自然触发 UI 能力，例如语音、表情、红包和转账。模型只负责输出固定格式，真正的执行、扣款、语音生成和渲染由前端代码控制。

## 开发原则

当前项目已经进入功能打磨阶段，后续开发建议遵守：

- 每次只改一个明确问题
- UI 修改不要顺手改聊天逻辑
- 特殊格式统一走 parser / directives
- 钱包交易必须保证账务一致
- 不要把 API、数据库、主题、语音逻辑混在一次改动里
- 优先保证项目能跑，再逐步整理代码结构

## 注意事项

1. API Key 默认保存在本地浏览器，请妥善保管。
2. 前端直连第三方 API 时，可能遇到 CORS 限制。
3. 钱包、红包、转账均为本地娱乐功能，不涉及真实支付。
4. 备份文件可能包含角色设定、聊天记录和 API 配置，请谨慎分享。
5. 推荐使用 Chrome、Edge 等现代浏览器。
