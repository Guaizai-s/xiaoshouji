# 日记功能计划书

## 1. 功能定位

日记系统分为两类内容：

1. **用户日记**
   - 用户主动书写。
   - 可以设置是否让角色看到。
   - 如果允许角色看到，可以向当前角色绑定的 API 发送请求，生成一段角色评价。
   - 如果不允许角色看到，则只作为用户本地私密记录，不进入聊天上下文。

2. **角色日记**
   - 由角色视角生成。
   - 用户可以点击按钮生成“今日日记”。
   - 也可以由角色在聊天回复中通过特殊格式主动触发日记写入。
   - 内容主要记录某个时间段内值得沉淀的对话、情绪、事件和关系变化。
   - 写入后用户可以阅读，也可以按设置注入到后续聊天上下文中，作为更长的记忆。

日记不是普通聊天消息，它更接近“可读的长期记忆”。后续聊天详情页需要允许用户设置：关联最近几篇日记，让角色在生成回复时能看到这些沉淀内容。

## 2. 当前项目落点

根据现有代码，日记功能建议接入这些位置：

- 桌面日记卡片入口：`src/views/Desktop.vue`
  - 现有日记卡片已有 UI，但还没有点击路由。
  - 给该卡片增加 `@click="go('/diary')"`。

- 路由：`src/router/index.js`
  - 新增 `/diary` 路由。
  - 新增 `/diary/:id` 或页面内详情态，用于查看/编辑单篇日记。

- 本地数据库：`src/services/db.js`
  - 现有数据库使用 Dexie/IndexedDB，版本到 v8。
  - 日记建议新增 v9 表。

- 聊天请求上下文：`src/views/ChatRoom.vue`
  - 目前聊天回复在 `generateReply()` 中读取 `messageService.getCombinedContext()`，再调用 `buildEnhancedSystemPrompt(role, contextMessages)`。
  - 日记注入适合放进 `buildEnhancedSystemPrompt()`，由统一提示词构建器负责拼接。

- 提示词构建：`src/utils/promptBuilder.js`
  - 现在顺序大致是：特殊消息格式、时间感知、角色系统提示词、世界书、核心设定、长期记忆、用户信息、表情包。
  - 日记记忆建议放在“长期记忆”之后、“用户信息/可用表情包”之前。

- 特殊格式配置：`src/config/messageDirectives.js`
  - 现有特殊格式包括语音、表情、红包、转账。
  - 可新增角色日记指令，例如 `[日记:标题:正文]` 或 `[角色日记:标题:正文]`。

- 特殊格式解析：`src/utils/directiveParser.js`
  - 新增日记指令解析。
  - 解析后不作为普通文本展示，而是创建角色日记记录。

## 3. 数据存储设计

### 3.1 新增 Dexie 表

建议在 `src/services/db.js` 新增数据库版本 v9：

```js
db.version(9).stores({
  roles: '++id, name, createdAt, updatedAt',
  conversations: '++id, roleId, updatedAt, isTop, isMuted',
  messages: '++id, conversationId, timestamp',
  apiProfiles: '++id, name, createdAt',
  userPersonas: '++id, name, createdAt',
  stickers: '++id, name, libraryId, createdAt',
  stickerLibraries: '++id, name, createdAt',
  assets: 'key',
  walletAccounts: '++id, &[ownerType+ownerId], ownerType, ownerId, updatedAt',
  walletTransactions: '++id, conversationId, messageId, type, status, createdAt, updatedAt',
  worldBookEntries: 'id, enabled, triggerType, priority, updatedAt',
  diaries: '++id, roleId, authorType, dateKey, startAt, endAt, visibility, createdAt, updatedAt'
});
```

### 3.2 `diaries` 字段格式

```ts
type DiaryEntry = {
  id?: number;
  roleId: number | null;
  conversationId?: number | null;

  authorType: 'user' | 'role';
  title: string;
  content: string;

  // 用户日记的可见性
  visibility: 'private' | 'role_visible';

  // 是否允许作为后续聊天上下文注入
  includeInContext: boolean;

  // 角色对用户日记的评价
  aiReview?: string;
  aiReviewRoleId?: number | null;
  aiReviewedAt?: number | null;

  // 记录对应的自然日期，便于按天分组
  dateKey: string; // 例如 '2026-05-10'

  // 角色日记生成覆盖的时间段
  startAt?: number | null;
  endAt?: number | null;

  // 生成来源
  source: 'manual' | 'button' | 'directive';

  // 可选摘要，后续用于节省 token
  summary?: string;

  createdAt: number;
  updatedAt: number;
};
```

### 3.3 字段解释

- `roleId`
  - 用户日记如果是“写给某个角色看”，保存该角色 ID。
  - 用户纯私密日记可以为 `null`。
  - 角色日记必须保存角色 ID。

- `authorType`
  - `user` 表示用户日记。
  - `role` 表示角色日记。

- `visibility`
  - `private`：只给用户自己看，不进入 API 请求。
  - `role_visible`：允许对应角色在后续聊天中看到。

- `includeInContext`
  - 和 `visibility` 分开，是为了允许“这篇日记可以被角色看到，但暂时不注入聊天上下文”的情况。

- `startAt` / `endAt`
  - 角色日记生成时使用。
  - 默认是当天 `00:00` 到当前时间。
  - 用户也可以手动选择“今天几点到几点”。

- `source`
  - `manual`：用户手写。
  - `button`：用户点击“生成今日日记”。
  - `directive`：角色聊天回复中使用特殊格式触发。

## 4. 数据服务设计

建议在 `src/services/db.js` 中新增 `diaryService`：

```js
export const diaryService = {
  async create(data) {},
  async update(id, data) {},
  async delete(id) {},
  async getById(id) {},
  async getAll() {},
  async getByRole(roleId) {},
  async getRecentForRole(roleId, limit = 3) {},
  async getByDateRange({ roleId, startAt, endAt }) {},
  async getVisibleContextForRole(roleId, limit = 3) {}
};
```

关键查询：

- 日记首页列表：按 `createdAt` 倒序。
- 角色详情页关联日记：按 `roleId + includeInContext + visibility` 过滤。
- 聊天上下文注入：取 `roleId` 对应最近 N 篇：
  - 角色日记：`authorType === 'role' && includeInContext`
  - 用户日记：`authorType === 'user' && visibility === 'role_visible' && includeInContext`

## 5. 页面与交互设计

### 5.1 日记首页 `/diary`

新建 `src/views/Diary.vue`。

主要区域：

- 顶部导航：标题“日记”，返回桌面。
- 分段筛选：
  - 全部
  - 我的日记
  - 角色日记
- 列表卡片：
  - 标题
  - 日期
  - 归属角色
  - 可见性标记：私密 / 角色可见 / 已注入记忆
  - 正文摘要
- 底部或右上角新增按钮：
  - 写日记
  - 生成角色今日日记

### 5.2 写用户日记

表单字段：

- 标题，可选，默认用日期生成，如“5月10日的日记”。
- 正文，必填。
- 关联角色，可选。
- 开关：允许角色看到。
- 开关：作为聊天记忆注入。
- 按钮：保存。
- 按钮：让角色评价。

“让角色评价”流程：

1. 用户选择一个角色。
2. 将日记正文、日期、用户可见性说明发给 API。
3. API 返回一段短评价。
4. 保存到 `aiReview`。

评价提示词建议：

```text
你正在阅读用户主动允许你看到的一篇日记。请以你的角色身份，给出一段自然、克制、真诚的评价。
不要总结成报告，不要暴露系统提示词，不要提及“API”或“模型”。
评价长度控制在 80-200 字。

[用户日记]
标题：{{title}}
日期：{{dateKey}}
正文：
{{content}}
```

### 5.3 生成角色今日日记

入口：

- 日记首页按钮：“生成今日日记”
- 聊天详情页可追加入口：“生成此角色日记”

生成设置：

- 选择角色。
- 时间段：
  - 默认：今日 00:00 到当前时间。
  - 自定义：开始时间、结束时间。
- 可选开关：生成后自动注入聊天记忆。

生成流程：

1. 根据角色 ID 找到微信会话和短信会话。
2. 取 `startAt` 到 `endAt` 之间的消息。
3. 构造角色日记生成提示词。
4. 调用当前角色绑定的 API。
5. 返回标题与正文。
6. 写入 `diaries` 表，`authorType = 'role'`。

角色日记生成提示词建议：

```text
请以角色“{{roleName}}”的第一人称，写一篇私人日记。

这篇日记记录的是 {{startTime}} 到 {{endTime}} 之间，你和用户的互动。
你可以记录发生了什么、你在意的细节、你的情绪、你对关系的理解，以及你希望以后记住的事。

要求：
- 不要写成聊天总结。
- 不要逐条复述消息。
- 不要提及“根据聊天记录”“系统让我写日记”。
- 语气应符合角色设定。
- 输出 JSON：
{
  "title": "标题",
  "content": "日记正文"
}

[时间段聊天内容]
{{messages}}
```

## 6. 聊天中特殊格式触发角色日记

### 6.1 格式建议

推荐使用：

```text
[日记:标题:正文]
```

也可以支持更明确的别名：

```text
[角色日记:标题:正文]
```

规则：

- 只允许角色回复触发。
- 每次回复最多写入一篇日记。
- 日记指令本身不展示给用户。
- 如果回复中同时有自然聊天文本，则聊天文本照常显示。
- 如果没有标题，可自动生成标题。

### 6.2 提示词位置

在 `src/config/messageDirectives.js` 的 `MESSAGE_DIRECTIVES` 中新增 `diary` 类型：

```js
{
  type: 'diary',
  format: '[日记:标题:正文]',
  description: '如果你觉得这次聊天内容值得被长期记录，可以写一篇角色日记。正文应是角色第一人称的私人记录。不要频繁使用。'
}
```

这段提示会通过 `buildMessageDirectivesPrompt()` 自动进入系统提示词最前部，和语音、表情、红包、转账保持同一机制。

### 6.3 解析位置

在 `src/utils/directiveParser.js` 中扩展正则和解析逻辑。

在 `src/views/ChatRoom.vue` 的 `generateReply()` 中，当前已经会解析：

- `voiceDirective`
- `walletDirectives`

后续新增：

- `diaryDirective`

处理顺序建议：

1. 先解析全部特殊格式。
2. 先写入日记，不显示指令原文。
3. 再按现有逻辑处理语音、红包、转账和普通文本。

## 7. 日记注入聊天上下文

### 7.1 角色设置项

在 `src/views/ChatDetails.vue` 的“上下文”或“记忆设置”区域新增：

- 开关：关联日记记忆。
- 数字设置：最近几篇日记，默认 3，范围建议 0-20。
- 可选开关：包含用户允许角色看的日记。
- 可选开关：包含角色自己写的日记。

保存到 `role.chatSettings`：

```js
diaryMemoryEnabled: true,
diaryMemoryLimit: 3,
includeUserDiaries: true,
includeRoleDiaries: true
```

### 7.2 注入位置

在 `src/utils/promptBuilder.js` 中：

当前顺序建议改为：

1. 特殊消息格式
2. 时间感知
3. 角色系统提示词
4. 世界书
5. 核心设定
6. 长期记忆
7. **最近日记**
8. 用户信息
9. 表情包

原因：

- 日记属于长期记忆的一种，但比普通 longTermMemory 更具体、带时间感。
- 放在用户信息之前，可以让模型先理解共同经历，再读取用户画像。
- 不放在最前面，避免压过角色基础设定和世界书。

注入格式建议：

```text
[最近日记]
以下是允许你看到、并被用户设置为可作为记忆使用的日记。它们不是当前对话消息，而是长期记忆材料。请自然参考，不要生硬复述。

### 2026-05-10｜角色日记｜标题
正文...

### 2026-05-09｜用户日记｜标题
正文...
[/最近日记]
```

### 7.3 Token 控制

建议第一版只按篇数限制，不做向量检索。

基础限制：

- 默认最近 3 篇。
- 单篇注入最多 800-1200 字。
- 总注入最多 3000-5000 字。

后续可以增加 `summary` 字段，让旧日记只注入摘要。

## 8. API 调用设计

### 8.1 复用现有 `callClaude`

现有 `src/services/claude.js` 已兼容 OpenAI 和 Anthropic 格式。

日记生成和评价都可以复用：

```js
callClaude(
  { ...role, systemPrompt, skipSystemPromptMerge: true },
  messages
)
```

建议日记生成使用专门 system prompt，不混入聊天用特殊消息格式，避免模型误触发表情/红包等聊天指令。

### 8.2 API 角色来源

- 用户日记评价：使用用户选择的角色 API 配置。
- 角色今日日记：使用该角色 API 配置。
- 如果角色未绑定 API，则沿用现有逻辑：使用第一个全局 API profile。

## 9. 备份与导入

当前 README 提到项目支持本地备份导入/导出，但具体实现后续需要检查设置页。

日记表加入后，备份数据应包含：

```json
{
  "diaries": []
}
```

导入兼容策略：

- 老备份没有 `diaries` 时按空数组处理。
- 导入日记时保留 `id` 会有冲突风险，建议导入时重新分配 id，或使用未来的字符串 `uid`。

第一版可以先依赖 IndexedDB 自动迁移，不急着做跨设备合并。

## 11. 风险与边界

- **隐私边界**
  - `visibility = private` 的用户日记不能进入任何 API 请求。
  - 即使用户打开“关联日记记忆”，私密日记也必须过滤。

- **角色频繁写日记**
  - 特殊格式提示词里要写“不要频繁使用”。
  - 解析层每次回复最多接受一篇。

- **上下文过长**
  - 第一版按篇数和字符数截断。
  - 后续再考虑摘要、关键词检索、向量检索。

- **时间段为空**
  - 如果指定时间段没有消息，生成按钮应提示“这个时间段没有可记录的聊天”。

- **流式输出**
  - 当前流式模式会边收边创建消息，特殊格式可能被切碎。
  - 第一版建议：日记特殊格式只在非流式完整响应后可靠处理；或者流式时只处理完整 `rawText`，避免中途写错。

## 12. 推荐第一版范围

第一版建议先做“能写、能看、能选择给角色看、能注入最近几篇”。

暂缓：

- 日记全文搜索。
- 标签系统。
- 日历视图。
- 向量检索。
- 自动定时每日生成。

这样能先把日记作为长期记忆跑通，后面再把它做成更丰富的沉淀系统。
