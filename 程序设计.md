项目背景与任务目标

我们需要开发一个高仿微信 UI 的本地 AI 多角色聊天应用。请根据最新的产品 UI 设计草图，重构现有的前端界面布局和路由逻辑。

核心修改指令（按底部四个 Tab 拆解）

1. 微信 (消息列表 - Chats)

顶部导航栏：移除右上角的 + 号按钮以及任何多余的局部功能键，只保留居中的“微信”标题。

列表交互：保留消息会话列表，点击任意角色列表项，即可进入该角色的聊天对话界面。

等待模型回复期间，上方角色名切换为“对方输入中”

2. 通讯录 (Contacts) —— 【核心重构区】

头部功能入口修改：

将原本的“新的朋友”修改为“新角色”。点击后，跳转到一个“创建角色”的表单页面。该表单需要包含以下输入/配置项（非常重要）：

API Key

System Prompt (系统提示词)

角色头像 (支持输入URL)

角色姓名

模型名称 (如 claude-4.6-sonnet)

接口类型 (提供选择器：OpenAI格式 / Anthropic格式)

保留“群聊”入口，点击事件暂时绑定一个空函数（后续开发）。

彻底删除“标签”、“公众号”等无关入口。

联系人列表：在功能入口下方，正常展示已经创建的 AI 角色列表。

3. 发现 (Discover)

界面极简处理：列表第一项仅保留“朋友圈”入口。

删除无用功能：删除“附近的人”、“扫一扫”、“看一看”等所有其他微信原生项。

功能留白：点击“朋友圈”后，暂时只需进入一个带有顶部导航栏的空白页面（UI 空壳即可，先不写后台逻辑代码）。

4. 我 (Me)

个人资料区：

顶部展示头像、昵称（示例图为“落”）和微信号。点击右侧的 > 箭头区域，需要支持修改个人头像和昵称。

保留“+ 状态”按钮，但仅作静态 UI 展示，点击事件留空。

功能列表区：

严格按照顺序保留以下列表项：服务、收藏、朋友圈、作品、表情、设置。

功能留白：上述所有的列表项，目前仅需完成前端 UI 绘制，点击事件全部做留白处理（绑定空函数），不需要写跳转逻辑。

技术规范要求

请优先搭建好静态 UI 和页面间的路由跳转骨架。

“新角色”的创建表单数据，请先做好本地状态收集（收集到一个 Object 中打印出来即可），底层数据库存储我们在下一步完成。

严格对齐微信的间距、字体大小和边框颜色。
